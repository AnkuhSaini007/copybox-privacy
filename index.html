1. Scope and introduction

This Privacy Policy applies to SaveCopy – Clipboard Manager & MultiCopy, a Chrome extension previously published under the name CopyBox.

SaveCopy helps users save selected or copied content, create reusable text snippets, organize items into categories, search stored content, and copy or paste saved items. This policy describes the data processed by those features.

2. Information SaveCopy handles

SaveCopy handles information only to provide its user-facing clipboard and snippet features. Depending on the features you use, this may include:

Snippets and templatesText, titles, descriptions, tags, HTML formatting, and other content you manually save.
Clipboard historyText and supported images copied on webpages when automatic clipboard history is enabled.
Source detailsThe page title and source URL associated with copied or selected webpage content.
Organization dataCategories, folders, colors, ordering, pins, favorites, locks, trash status, and recent-use timestamps.
PreferencesTheme, density, font size, default popup view, cleanup period, and other extension settings.
Local backupsJSON files that you explicitly export or import using the extension's backup controls.
Automatic clipboard history: In the current version, this feature may be enabled by default. You can disable it at any time from SaveCopy Settings.

3. How SaveCopy uses information

SaveCopy uses locally handled information only to provide the extension's core functions, including:

  • Saving and displaying clipboard history and manually created snippets.
  • Searching, filtering, organizing, pinning, favoriting, locking, and deleting saved items.
  • Providing Recent, General, Clipboard, All, and custom category views.
  • Copying saved content back to the system clipboard or pasting it into a supported webpage field after a user action.
  • Detecting duplicate clipboard entries.
  • Applying user-selected automatic cleanup settings.
  • Creating or restoring a local JSON backup when requested by the user.

SaveCopy does not use saved content for advertising, profiling, credit decisions, or unrelated purposes.

4. Local storage and retention

SaveCopy stores extension data in Chrome's local extension storage on the user's device. The extension does not require an account and does not upload saved snippets, clipboard history, images, categories, or settings to an external SaveCopy server.

Clipboard cleanup

The current default cleanup period is 30 days for unpinned Clipboard items. Users can choose 7 days, 30 days, 90 days, or disable automatic cleanup. Pinned clipboard items are not removed by automatic cleanup.

Trash and deletion

Deleted items may first move to the extension's Trash, where they can be restored or permanently removed. Users can also empty the Trash. Uninstalling SaveCopy removes extension storage according to Chrome's extension data behavior.

JSON backup

When you export a JSON backup, the file is created only after your action and saved through your browser's download process. SaveCopy does not automatically upload that file. You are responsible for storing exported backups securely.

5. Data sharing and external transfers

SaveCopy does not sell user data, share saved content with advertisers or data brokers, or transmit clipboard history to a SaveCopy cloud service.

The current extension does not use Google Drive synchronization, cloud authentication, external analytics, advertising SDKs, or remote executable code.

Content is transferred to a webpage only when you perform a supported paste action. Content is written to the system clipboard only when you choose a copy action or use a supported SaveCopy command.

6. Chrome permissions

SaveCopy requests Chrome permissions only to provide its declared features:

PermissionWhy it is used
storageStores snippets, clipboard history, categories, settings, pins, favorites, locks, trash items, and other extension data locally.
contextMenusAdds a user-initiated right-click option for saving selected webpage text into SaveCopy.
activeTabAllows SaveCopy to access the current tab after a user invokes a supported save or paste action.
scriptingReads selected text or inserts saved text into the focused field after a user-initiated command.
clipboardWriteCopies a selected SaveCopy item back to the system clipboard.
<all_urls>Allows the clipboard-history content script and supported save/paste features to work on regular webpages. Chrome internal pages and other restricted pages remain unavailable.

7. Your choices and controls

You can control your SaveCopy data directly from the extension. Available controls include:

  • Enable or disable automatic clipboard history.
  • Change or disable the automatic cleanup period.
  • Pin items so automatic clipboard cleanup does not remove them.
  • Edit, delete, restore, or permanently remove saved items.
  • Empty the Trash.
  • Export data to a local JSON file and import a previous backup.
  • Remove extension data by uninstalling SaveCopy.

8. Security and sensitive information

SaveCopy is designed to keep saved content in Chrome's local extension storage. Security also depends on your device, browser profile, operating-system account, and access controls.

SaveCopy does not claim that locally stored content or exported JSON files are independently end-to-end encrypted by the extension. Avoid storing passwords, authentication tokens, payment details, private keys, health records, or other highly sensitive information in any clipboard manager.

9. Children's privacy

SaveCopy is a general productivity tool and is not designed to knowingly collect personal information from children. The extension does not require an account or request a user's age.

10. Changes to this policy

This policy may be updated when SaveCopy's features, permissions, storage behavior, or data practices change. Material updates will be posted on this page with a revised “Last updated” date.

SaveCopy's use of information received from Chrome APIs will follow the Chrome Web Store User Data Policy, including applicable Limited Use requirements.
